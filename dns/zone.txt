; DNS zone file for shanesparks.com — BIND format, importable via
; Cloudflare dashboard: DNS > Records > Import and Export.
;
; The domain is the single irreplaceable asset: registrar 2FA on,
; auto-renew on, registered under an email that does not depend on
; this domain. Recreate this zone at any DNS host in minutes.
;
; NOT in this file (cannot be expressed as ordinary records):
;   1. The apex (shanesparks.com -> Worker): Cloudflare creates and
;      manages it when the custom domain is attached to the Worker
;      (Workers & Pages > shanesparks-com > Settings > Domains & Routes).
;   2. The www redirect rule: Rules > Redirect Rules >
;      www.shanesparks.com/* -> https://shanesparks.com/$1 (301).
;
; After import, check proxy status in the dashboard:
;   www    -> proxied (orange cloud), so the redirect rule fires
;   mirror -> DNS only (gray cloud) — GitHub Pages must see the CNAME
;             directly to verify the domain and issue its certificate

$ORIGIN shanesparks.com.
$TTL 3600

; www resolves to the apex (Cloudflare flattens the CNAME)
www     IN  CNAME  shanesparks.com.

; GitHub Pages mirror (hot standby). Requires the CNAME line in
; scripts/deploy.ps1 uncommented and the custom domain set in the repo's
; Pages settings. If Cloudflare ever drops us: repoint the apex here.
mirror  IN  CNAME  sparktree.github.io.

; CAA: only these CAs may issue certificates for the domain.
; Covers Cloudflare Universal SSL (letsencrypt.org / pki.goog / ssl.com)
; and GitHub Pages (letsencrypt.org).
@       IN  CAA    0 issue "letsencrypt.org"
@       IN  CAA    0 issue "pki.goog"
@       IN  CAA    0 issue "ssl.com"
